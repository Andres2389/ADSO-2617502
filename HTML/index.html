iniciando con git

git es un control de versiones que permite a equipos a desarrolladores trabajar de maneras remota sobre un mismo proyecto. permite hacer contribuciones de codigo de manera local y luego de un repositorio remoto

PRICIPALES COMANDOS CON GIT

  • git initfunciona para inicializar un proyecto en git
  • git config -- global user.email "email" funciana para identificar y configurar globalmente el email con el repositorio local
  • git config -- global user.name "name" para identificar y configurar globalmente el nombre de usuario con el reposistorio local
  • git config global user.email es para verificar el email agregado
  • git config global user.name es para verificar el nombre agregado
  • git add.:funciona para agregar todos los archivos modificados y nuevos en el reposistorio local
  • git status permite verificar el estado del repositorio local
  • git commit -m "comentario":: permite guardar los cambios realizados en el repositorio local con un mensaje resumido.
  • git log muestra el historial de cambios en el repositorio local
  • git branch: muestra la rama del repositorio local
  • git checkout -b nombre-rama cambiar nombre de la rama en un repositorio local
  • git checkout -- .:deshacer los cambios en un archivo no confirmado.
  • git remote add origin el repositorio local se agrega a un repositorio remoto como GITHUB
  • git push -u origin main:enviar los comits del repositorio local a repositorio remoto de GITHUB.
